Hexo 升级

发表于 2020-02-18

准备

查看 node.js 和 npm 的版本

将其升级到最新版本

升级 npm

报错

根据提示 Please try running this command again as root/Administrator 可知是权限不够，以管理员权限运行：

查看版本，已升级到 6.13.7, That’s OK.

升级 Node

sudo npm cache clean -f //清除nodejs的cache：
sudo npm install -g n //使用npm安装n模块
npm view node versions // node所有版本
sudo n latest // 升级到最新版本
sudo n stable // 升级到稳定版本
sudo n xx.xx // 升级到具体版本号

首先清除 Node.js 的缓存，然后使用 npm 安装 Node.js 的多版本管理器 n 模块，最后升级到稳定版本。

查看版本，已安装 12.16.1，That’s OK.

升级 Hexo

先全局升级hexo-cli：-g表示全局升级。hexo本身是一个静态博客生成工具，具备编译Markdown、拼接主题模板、生成 HTML、上传 Git 等基本功能。hexo-cli能够将这些功能封装为命令，提供给用户通过hexo server / hexo deploy等命令调用的模块。CLI = Command Line Interface命令行界面。

查看 hexo 当前版本

使用 npm 自动更新

  1. 升级 hexo-cli

  2. 使用npm-check 检查依赖模块的更新

    可以看到，依赖模块的更新信息如下：

    ![](/Users/liger/Nustore Files/我的坚果云/个人博客/202002191856.png)

  3. 使用 npm-upgrade 来更新 hexo 依赖模块

    这时会有个选择菜单供你选择是否升级依赖包：

    全部 Yes.

    再使用 npm-check 查看，发现已全部更新。

    或者查看 hexo 目录下的 package.json 文件，确认依赖包已全部更新。

  4. 更新 hexo

    查看 hexo 版本，已更新。

手动更新

  1. 查看有哪些需要更新

  1. 进入 Blog 目录，修改 package.json 文件，将版本号改为最新

    {
    “name”: “hexo-site”,
    “version”: “0.0.0”,
    “private”: true,
    “hexo”: {

    "version": "4.2.0"

    },
    “dependencies”: {

    "hexo": "^4.2.0",
"hexo-deployer-git": "^2.1.0",
"hexo-generator-archive": "^1.0.0",
"hexo-generator-category": "^1.0.0",
"hexo-generator-index": "^1.0.0",
"hexo-generator-tag": "^1.0.0",
"hexo-renderer-ejs": "^1.0.0",
"hexo-renderer-marked": "^2.0.0",
"hexo-renderer-stylus": "^1.1.0",
"hexo-server": "^1.0.0"

    }
    }

    然后更新即可。
